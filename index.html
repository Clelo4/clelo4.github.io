指针与数组

1. 引言

在C++中，数组是一个独特的存在，它和指针一样神奇。C++不像支持第一数组类型如整型获取甚至结构体那样支持数组。

2. 数组的本质

数组名是个独特的存在，在C和C++中，大多数表达式把数组名转换成指向它第一个元素的指针。

3. 数组和指针的差别

在大多数时候，数组名表现为一个指针，可以像指针一样进行操作。除了在声明中或当一个数组名是sizeof运算符或&运算符的操作数之外，编译器总是把数组名解释成指向它的第一个元素的指针。

数组作为参数，实际上是传递了指向了数组第一个元素的指针，因此，数组参数本质上是指针参数，在函数内部对一个数组参数进行sizeof操作是不会输出数组的长度，只会输出该参数的指针长度。

what does “default” mean after a class’ function declaration?

class C {
    C(const C&) = default;
    C& operator=(const C&) = default;
    ~C() {}
}
  • It means that you want to use the compiler-genearated version of that function, so you don’t need to specify a boyd.

CSS定位模型

Posted on 2019-02-01 | In CSS

1. 相对定位

相对定位可以使得元素相对它的起点偏移一段距离，设置left和top的值，元素可以偏移到指定位置。

相对定义后的元素依然占据文档流的空间，仍然会影响到其他元素的布局，只是显示会偏离原来的位置。因此，偏移后的元素可能会覆盖其他元素。

2. 绝对定位

绝地定位使元素脱离文档流，不占据文档流空间，普通文档流中其他元素的布局就像绝对定位的元素不存在一样。绝对定位的元素的位置是相对于离它最近的那个已定位的父元素确定的。如果没有已定位的父元素，则相对于body进行定位。

z-index属性控制绝对定位的框的位置，z-index值越高，框在栈中的位置就越高。

3. 固定定位

固定定位是绝对定位的一种，区别在于固定定位是相对于视区(viewport)进行定位的，这样我们可以将一个元素框固定在视区窗口中。

4. 浮动定位

浮动定位一种相对复制的定位模型。浮动框可以左右浮动，直到它遇到它的包含框或遇到其他浮动框的边缘。浮动框不存在于文档流，一般情况下，文档流中的普通框无视浮动框的存在（特殊情况除外）。

浮动框带来的问题：
浮动定位让元素脱离了文档流，实际上浮动框还是会影响到文档流中其他框的布局。

调度

三级调度机制：高级调度、中级调度和低级调度

1. 高级调度（作业调度）

作业调度决定外存上后备队列上哪些作业可以调入内存，并为它们创建进程、分配必要的资源，然后将新创建的进程插入到就绪队列，准备运行。

作业调度决定:

1) 接纳多少个作业
2) 接纳哪些作业

2. 低级调度（进程调度）

进程调度用于决定就绪队列中的哪些进程可以获得cpu，然后在有分派进程执行把cpu分配给进程的具体操作。

进程调度有两种方式：

1) 抢占式
2) 非抢占式

3. 中级调度

挂起和激活，存储器管理中的对换功能。主要目的是为了提高内存的利用率和系统的吞吐量。

面向用户的调度性能准则

  1. 周转时间：作业从提交（作业调度进入内存）到完成（得到结果）所经历的时间。包括在等待队列、cpu上执行、就绪队列和阻塞队列中等待时间、结果输出等待。
    • 平均周转时间
    • 平均带圈周转时间 = 周转时间/CPU执行时间
  2. 响应时间：用户输入一个请求到系统给出首次响应的时间
  3. 截止时间：开始截止时间和完成截止时间
  4. 公平性
  5. 优先级

面向系统的调度性能准则

  1. 吞吐量：单位时间内所能完成的作业数。
  2. 处理机利用率：— 大中型主机
